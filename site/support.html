From Developer to Developer

Open Issue

For bug reports, feature requests, and troubleshooting details, use GitHub Issues: github.com/KrystofSlama/RepoStudio/issues

Before opening an issue

  • Confirm you are on the latest app version.
  • Check if an existing issue already matches your problem.
  • Test with a small sample repository if possible.

Include in your report

  • macOS version and app version.
  • Steps to reproduce the problem.
  • Expected behavior vs actual behavior.
  • Screenshot if the issue is visual.

Common troubleshooting

  • Make sure Git is installed and available in your PATH.
  • Verify the repository opens correctly in Terminal with git status.
  • If markdown rendering looks wrong, test with a minimal markdown file first.

Security reports

If you discover a security issue, include "Security" in the issue title and avoid posting sensitive personal data in public threads.

Response policy

RepoStudio is an independent project, and support is provided on a best-effort basis.
