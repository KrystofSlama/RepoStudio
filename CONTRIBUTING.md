# Contributing to RepoStudio

Thank you for your interest in contributing.

## Ways to Contribute

- Report bugs or UX issues.
- Suggest improvements.
- Submit bug fixes.
- Submit new features that fit the project scope.

## Development Setup

1. Fork the repository.
2. Clone your fork.
3. Open `macOS/RepoStudio/RepoStudio.xcodeproj` in Xcode.
4. Build and run the `RepoStudio` scheme.

## Pull Request Guidelines

1. Create a focused branch for each change.
2. Keep pull requests small and easy to review.
3. Include a clear description of what changed and why.
4. Add or update tests when appropriate.
5. Confirm the project builds before opening the PR.

Be respectful, constructive, and collaborative in issues and pull requests.
